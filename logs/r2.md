  The IBRL checkpoint got 0/50 correct, so all rollouts are "incorrect." Key observations:
  - MBE is uniformly ~1.95 everywhere — the model has maximized representation diversity to saturation
  - Log-probs decrease monotonically from -0.16 to -0.53 — confidence drops throughout generation
  - Completion length: 492 (vs GRPO's 274/316) — the model generates maximally long, low-confidence text
  - All completions hit the 512 token limit, confirming the model is just producing high-entropy text

  Key Takeaways

  1. MBE is a meaningful discriminator: On the working GRPO model, correct rollouts have consistently higher MBE (+0.06 on average). This validates MBE as a useful
  metric for distinguishing good reasoning.
  2. Confidence separates correct from incorrect: The most discriminative region is 65-85% through generation, where correct rollouts are ~0.10 nats more confident.
  3. Correct rollouts are shorter: 274 tokens vs 316 — successful reasoning is more concise.
  4. MBE as a regularizer still fails: Despite MBE being a good diagnostic metric, using it as a training objective (whether minimize or maximize) causes collapse. The
   fundamental issue is that MBE as a loss term dominates the gradient and fights the task reward signal.
