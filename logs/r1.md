Detailed IBRL Collapse Analysis

  The death spiral (3 phases)

  Phase 1 — MBE dominates early (steps 1–20)
  - MBE loss starts at 0.80 and is immediately the strongest gradient signal
  - MBE contribution is ~13% of total loss, but it's the only consistent direction — the policy loss is noisy (depends on
  reward variance across generations)
  - The model rapidly compresses hidden representations to minimize MBE
  - Entropy jumps 0.6 → 1.6 in just 10 steps — the model's output distribution is already flattening

  Phase 2 — Reward signal dies (steps 20–30)
  - frac_reward_zero_std hits 1.0 at step 30 and stays there permanently
  - This means: in every group of 8 generations, ALL get the same reward (0.0). GRPO computes advantages as reward -
  group_mean, so all advantages are exactly zero
  - Zero advantages = zero policy gradient. The GRPO loss term becomes a no-op
  - MBE is now the ONLY loss driving updates, pushing representations further into collapse
  - Token entropy explodes to 4.3 (near-uniform over vocabulary)

  Phase 3 — Dead equilibrium (steps 30–200)
  - MBE itself collapses to ~0.002 — representations are completely degenerate
  - Grad norm drops to 0.003–0.007 — essentially no learning
  - The model outputs near-random tokens, which always get reward 0, which means zero advantage, which means no policy
  gradient to recover
  - This is a stable fixed point — there's no gradient to escape

  Why GRPO survives

  GRPO maintains frac_reward_zero_std between 0.2–0.6 throughout training. Some groups always have reward variance (some
  generations get the answer right, others don't), so there's always a policy gradient signal pushing the model to produce
  correct answers more often.

  Root cause

  The MBE regularizer creates a competing objective that wins the race in early training:

  GRPO wants: concentrate probability on correct answers (lower entropy)
  MBE wants:  compress hidden representations (also lower entropy... of representations)

  But crushing representation entropy destroys the model's ability to distinguish between tokens, which makes ALL outputs
  equally bad, which kills the reward signal, which removes the GRPO gradient, which lets MBE run unopposed.

  ------------------------------------------------
  I think we need "bottleneck MBE regularization" instead of mean MBE regularization, but this experiment suggests including MBE into loss 
  might not be a great idea, likely due to the instability issue in RL
  training. The gradient signal from MBE is too "consistent" and it overwhelms
  the policy gradient signal. 

  It's then worth tracking the MBE & CE of "successful" rollout v.s. "unsuccessful" rollout, to see if MBE can help to "trim rollout".
